# Day-13 – Need of Client-Server Architecture – Core Model

This folder contains concise notes explaining **why client-server architecture emerged**, **how it works**, and **why it scales**.

## Contents
- `NOTES.md` – Conceptual notes with flow and key insights

## Key Takeaway
Client-server architecture centralizes trust and rules on the server while keeping interaction flexible on the client. This separation enables scalability, consistency, and predictable behavior in large systems.
